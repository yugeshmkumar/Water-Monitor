## KEY ELECTRICAL CHARACTERISTICS

### Soft-Start Current Limiting
- Polyfuse F1: Holds at 1.1A, trips at 1.35A
- P-FET Q1: RDSon = 50mΩ, turns on slowly via R5
- Inrush: < 1A over first ~100ms (soft ramp, not hard click)

### Voltage Divider (5V → 3.3V)
- R3 + R4 creates 1.67V from 5V input when terminated with GPIO20 input
- Actually: V_out = 5V × 20k/(10k+20k) = 3.33V ≈ max GPIO rating
- Impedance: R_th = 10k || 20k = 6.67kΩ (very low loading on GPIO20)

### Decoupling Network Impedance
- Target: < 100 mΩ @ 100kHz
- C1 (680µF): Dominates DC-10Hz, ESR ≈ 30mΩ
- C2, C3 (100µF): Provide 10-100Hz, ESR ≈ 20mΩ each
- C4, C5, C6 (100nF): Handle 100Hz-1GHz, ESR ≈ 5mΩ each
- Total: Cascaded impedance provides smooth response across all frequencies

### Ferrite Bead Effectiveness
- 1kΩ @ 100MHz means:
  - @ 10MHz: ~300Ω impedance (still good isolation)
  - @ 1MHz: ~100Ω impedance (some attenuation)
  - @ 100kHz: < 50Ω impedance (minimal effect)
- Optimized for 40 kHz sensor harmonics (2nd harmonic at 80 kHz)

## SIGNAL INTEGRITY VERIFICATION

### Voltage Levels (Operating)
- +5V rail: 4.75V – 5.25V (under load with polyfuse)
- GPIO21 (ESP32 output): 3.0V – 3.6V (logic high)
- GPIO20 divider input (5V sensor): 3.0V – 5.5V (depends on sensor)
- GPIO20 (ESP32 input): 0.0V – 3.3V max (safe after divider)

### Timing
- UART baud: 9600 bps = 104 µs per bit
- Frame time: 4 bytes = ~4.3 ms reception
- Sensor response latency: 0-120 ms (specified in SR04M-2 datasheet)
- Polling interval: 5 seconds (firmware configured)
